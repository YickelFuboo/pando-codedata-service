# 测试文件说明

## 文件用途说明

### 1. `conftest.py` - Pytest配置文件
**用途**: pytest的配置文件，用于设置测试环境的路径和公共配置
- 配置Python路径，确保测试可以正确导入模块
- 所有测试文件都会自动加载此配置

### 2. `test_models.py` - 数据模型结构测试
**用途**: 验证AST分析结果的数据模型结构是否正确
- 测试 `FileInfo`、`FunctionInfo`、`ClassInfo` 等数据模型
- 验证模型字段是否存在
- **不测试AST分析功能，只测试数据结构**

### 3. `test_file_analyzer.py` - 语言检测测试
**用途**: 测试文件分析器的语言检测功能
- 验证能否正确识别Python、Java、Go等语言
- 测试未知语言的处理

### 4. `test_python_ast_analysis.py` - Python AST分析核心测试 ⭐
**用途**: **核心测试文件**，验证Python AST分析功能的正确性
- 函数提取：函数名、参数、返回类型、文档字符串
- 类和方法提取：类名、方法名、方法类型
- 导入语句解析：绝对导入、相对导入
- 继承关系识别
- 函数调用关系
- 异步函数、复杂签名等

### 5. `test_java_ast_analysis.py` - Java AST分析测试
**用途**: 验证Java AST分析功能
- 类提取
- 继承关系
- 接口实现
- 导入语句

### 6. `test_go_ast_analysis.py` - Go AST分析测试
**用途**: 验证Go AST分析功能
- 函数提取
- 结构体提取
- 接口实现

### 7. `test_c_ast_analysis.py` - C AST分析测试
**用途**: 验证C AST分析功能
- 函数提取
- 结构体提取
- 指针函数
- 头文件包含

### 8. `test_cpp_ast_analysis.py` - C++ AST分析测试
**用途**: 验证C++ AST分析功能
- 类提取
- 继承关系
- 模板
- 命名空间
- 运算符重载

### 9. `test_folder_ast_analysis.py` - 文件夹分析测试
**用途**: 验证文件夹AST分析功能
- 项目结构分析
- 隐藏目录排除（.git, __pycache__等）
- 多语言文件支持
- 相对路径计算
- 嵌套文件夹结构

### 10. `test_integration.py` - 集成测试
**用途**: 测试完整的AST分析流程
- 创建完整的项目结构（包含多个文件、文件夹）
- 验证文件分析、文件夹分析的协同工作
- 验证导入关系、类提取等功能的集成

## 测试源文件目录

### `test_sources/` - 测试源文件目录
**用途**: 存放用于测试的各种语言的示例源码文件
- `python/` - Python测试源文件
- `java/` - Java测试源文件
- `go/` - Go测试源文件
- `c/` - C测试源文件
- `cpp/` - C++测试源文件

这些源文件可以作为测试用例的输入，验证AST分析器能否正确解析各种代码结构。

## 测试文件组织原则

1. **按功能分类**: 每个语言有独立的测试文件
2. **按层次分类**: 
   - 单元测试：单个功能点（如函数提取）
   - 集成测试：完整流程（如项目分析）
3. **测试源文件分离**: 测试代码和测试数据分离，便于维护
